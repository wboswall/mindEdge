													<me-users-mobile-app-prompt></me-users-mobile-app-prompt> <h3>What is Responsive Design?</h3>

<p><a role="button" tabindex="0" class="glossary-alert" id="glossTerm0" data-term="responsive design" name="responsive design" title="The use of CSS and HTML to resize, hide, shrink, enlarge, or move content so that it looks good on any device with any size screen">Responsive design<span class="screenreader">*</span></a><span style="display:none;">&nbsp;The use of CSS and HTML to resize, hide, shrink, enlarge, or move content so that it looks good on any device with any size screen&nbsp;</span> is a term coined in 2010 by web developer Ethan Marcotte. Leveraging grid-based layouts, media queries, and images that resize, Marcotte created a new technique for building websites with a single code base that would display content in a readable manner, compatible with screens of any size. The technique was first described on the web magazine <a href="https://alistapart.com/article/responsive-web-design" target="_blank">A List Apart</a>.</p>

<p>In 2010, the iPhone and Android operating systems were just a few years old; the iPad had only been in existence for five months when Marcotte's article was published. Before responsive design, it was common for developers to create two entirely different website layouts, one for phones and one for desktop computers. Typically this approach was created in one of several ways:</p>

<table class="default_tbl4">
  <tr>
    <th colspan="2">Early Methods used to Design for Mobile and Desktops</th>
  </tr>
<tr>
    <th style="width:40%">Content Management Systems</th>
  <td style="width:60%"><p>If a content management system like WordPress was used, the same content would load into entirely separate templates or themes to display on the device of choice.</p>
    <p><b>Main Drawback:</b> While website maintenance was a little easier with the same content loading on both sites, maintaining the HTML and CSS was time-consuming and challenging.</p></td>
</tr>
<tr>
<th style="width:40%">Two Code Bases: Mobile &amp; Desktop</th>
  <td style="width:60%"><p>Some developers would use a tool like jQuery Mobile to create one website for phones and a second for desktops. Unfortunately, using jQuery Mobile without a database meant that two versions of site content were created, since these HTML, CSS, and JavaScript files were completely separate for each platform.</p>
    <p><b>Main Drawback:</b> There was no way to share content between devices. Furthermore, the website addresses for a mobile site vs. a desktop site were different, which was confusing for many users.</p>
</td>
</tr>
<tr>
<th style="width:40%">Custom Solution</th>
  <td style="width:60%"><p>Developers wrote a completely custom solution</p>
    <p><b>Main Drawback:</b> This approach usually resulted in two code bases again.</p></td>
</tr>
</table>

<p>Responsive design, however, addressed all of these problems:</p>

<div class="big yellow highlightbox">
  <center><b>Responsive Design</b></center>

  <ul>
<li>By using a single code base for website layout, website maintenance was simplified for developers.</li>
<br>
  <li>Only one version of the content needed to be written, which could then be displayed on any platform at an appropriate screen size.</li>
<br>
<li>Designs were not limited to phone (narrow) vs. desktop (wide). With the introduction of the iPad and the rapid proliferation of different sized devices that followed shortly after, designs needed to adapt to screen sizes that hadn't yet been defined or invented. Only responsive design could do this.</li>
</ul>
</div>

<h4>Where is responsive design used today?</h4>

<p>Responsive design is available in off-the-shelf coding frameworks like Bootstrap, Foundation, and UIkit, among many others. Individuals may also write code to create a responsive website.</p>

<p>Responsive design is compatible with many web design technologies, including content management systems like WordPress, JavaScript-based sites using React or Node.js, websites using databases, and server-side technologies like PHP or Python; and static websites built with HTML and CSS. Responsive design may be used with any version of a website.</p>
