													<me-users-mobile-app-prompt></me-users-mobile-app-prompt> <h3>Understanding Media Queries</h3>

<p><a role="button" tabindex="0" class="glossary-alert" id="glossTerm2" data-term="media queries" name="media queries" title="Enable webpage content to adapt to different screen sizes and resolutions. They are a fundamental part of responsive web design and are used to customize the appearance of websites for multiple devices">Media queries<span class="screenreader">*</span></a><span style="display:none;">&nbsp;Enable webpage content to adapt to different screen sizes and resolutions. They are a fundamental part of responsive web design and are used to customize the appearance of websites for multiple devices&nbsp;</span> are an essential part of responsive design. Using media queries, designers can apply different styles and adapt layouts for different screen sizes and resolutions. The syntax for a media query is listed below.</p>

<div class="big yellow highlightbox centerdiv">
<code>
        @media not|only mediatype and (mediafeature and|or|not mediafeature) {
           CSS code {
             ...
       }
        }
</code>

<p>The @media syntax consists of:</p>

<ul>
  <li>A media type &mdash; such as screen (used for computer screens, tablets, or mobile phones), print (used for printers), or all (used for all media devices &mdash; this is the default if nothing is defined)</li>
  <li>One or more media features that define the parameters &mdash; such as screen width or resolution</li>
  <li>CSS code placed inside curly brackets &mdash; &#123;  CSS code  &#125; &mdash; that define style rules</li>
</ul>
</div>

<p>A media query is a logical expression that is resolved to be either true or false. The result of a media query is true if the media query matches the type of device the document is being displayed on. If this is the case, then all the related CSS styles in the media query are applied to the target device.</p>
  
  <p>In other words, the @media at-rule applies part of a CSS style sheet based on the result of a media query. This CSS construct is a way of testing for certain conditions in your CSS, such as:</p>

<ul>
  <li>How wide the browser window is, including a minimum, maximum, or a range of sizes</li>
  <li>Whether you are using a screen display or preparing a web page for print</li>
  <li>Whether the screen is oriented in portrait or landscape displays</li>
  <li>Or a combination of these conditions</li>
</ul> 

<p>Media queries may be placed directly into a webpage's HTML or included in a separate .CSS file that is referenced by the webpage. There is no limit on the number of media queries a .CSS file may contain. A designer can place as many media queries as the design requires, and may combine media queries to get the exact layout results desired.</p>

<h4>Media Queries: Desktop-First versus Mobile-First Approach</h4>

<p>Like all modules within a cascading style sheet, the ones that appear further down the list override the ones above them. <a role="button" tabindex="0" class="glossary-alert" id="glossTerm1" data-term="desktop-first" name="desktop-first" title="Design approach where the initial design is structured for larger screen sizes first, and then scaled down through the use of media queries for smaller screen sizes">Desktop-first<span class="screenreader">*</span></a><span style="display:none;">&nbsp;Design approach where the initial design is structured for larger screen sizes first, and then scaled down through the use of media queries for smaller screen sizes&nbsp;</span> designs for large screen layouts first, and then uses media queries to override any settings for smaller devices. The opposite, known as <a role="button" tabindex="0" class="glossary-alert" id="glossTerm3" data-term="mobile-first" name="mobile-first" title="Design approach where the initial design is structured for small screen sizes first, and then scaled up through the use of media queries for larger screen sizes">mobile-first<span class="screenreader">*</span></a><span style="display:none;">&nbsp;Design approach where the initial design is structured for small screen sizes first, and then scaled up through the use of media queries for larger screen sizes&nbsp;</span>, defines styles for mobile devices first, followed by media queries to override any settings for larger devices. "Mobile-first" development will be discussed in more depth in a later assignment in this course.</p>

<h4>Example:</h4>

<p>Consider the following CSS. In this example, the media query starts with an @media statement, followed by a parameter to test. We are testing to see if the maximum width is 400px. This is followed by a curly brace, which must be closed at the end of the media query.</p>

function toggleType(section, type){
    const sectionContainer =$(`#codingSection-${section}`);
    const htmlSection = sectionContainer.find(`#html`);
    const cssSection = sectionContainer.find(`#css`);
    const jsSection = sectionContainer.find(`#js`);
    if(lastType[section] === type){
        return;
    }
    lastType[section] = type;
    switch (type) {
        case 'html':
            htmlSection.toggleClass('active');
            cssSection.removeClass('active');
            jsSection.removeClass('active');
            break;
        case 'css':
            cssSection.toggleClass('active');
            htmlSection.removeClass('active');
            jsSection.removeClass('active');
            break;
        case 'js':
            jsSection.toggleClass('active');
            cssSection.removeClass('active');
            htmlSection.removeClass('active');
            break;
    }

function updateFrame(section, html) {

function replaceStyles(html, cssBlock){
    const regex = /(<link\b.+href=")(?!http)([/styles.css]*)(".*>)/gm;
    const cssContent = cssBlock.getContent();
    const content = `<style> ${cssContent} </style>`;
    html= html.replace(regex, content);
    return html;
}

function replaceJsScripts(html, jsBlock) {
    const regex = /(<script\b.+src=")(?!http)([/scripts.js]*)(".*>)/gm;
    const start ='<';
    const end=">";
    const jsContent = jsBlock.getContent();
    const content = `${start}script${end} ${jsContent} ${start}/script${end}`;
    html= html.replace(regex, content);
    return html;
}
