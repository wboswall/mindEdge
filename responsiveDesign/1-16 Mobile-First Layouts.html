													<me-users-mobile-app-prompt></me-users-mobile-app-prompt> <h3>What is Mobile-First Layout?</h3>

<img src="https://cdn-d.mindedgeonline.com/1678/drafting-up-a-mobile-deisng-for-a-website.jpg" alt="Drawing the wireframe of a website in mobile format." class="img-center" style="max-width:70em;">

<p>When it comes to mobile devices, web designers are challenged to consider a number of constraints when it comes to designing webpages. For one, mobile devices generally have less bandwidth, processing power, and memory than their desktop counterparts. Additionally, mobile devices have smaller viewports. Websites that are designed with larger screens or processing power don't offer an optimum user experience when viewed on a mobile device as they often load slowly and lag in terms of performance. With the consumption of mobile media at an all-time high &mdash; and growing &mdash; <a role="button" tabindex="0" class="glossary-alert" id="glossTerm1" data-term="mobile-first" name="mobile-first" title="Design approach where the initial design is structured for small screen sizes first, and then scaled up through the use of media queries for larger screen sizes">mobile-first<span class="screenreader">*</span></a><span style="display:none;">&nbsp;Design approach where the initial design is structured for small screen sizes first, and then scaled up through the use of media queries for larger screen sizes&nbsp;</span> design strategy has become a principle in website design and development.</p>   

<p>Recall, the opposite approach &mdash; desktop-first &mdash; designs for a desktop or large screen layouts first, and then uses media queries to override any settings for smaller devices.</p>
  
<p>As a design strategy, mobile-first dictates that a website should be designed for the smallest screen first. In other words, the mobile layout is considered the default layout upon which to build layouts for other devices and/or screen dimensions. The reasons for this are twofold:</p>

<dl class="jquery_accordion">

  <dt>1. Website Design Planning</dt>
  <dd>
	<p>The principles of user experience (the process of planning a website and creating designs) guide designers to consider mobile devices first when planning the website's messaging. The key focus is that the most important information should be clearly identified. It challenges designers to consider their designs in light of the following question:</p>
	
	  <blockquote class="quotationblock">
<p>If there is no room on the mobile screen for certain information, why include it on the website at all?</p>
</blockquote>
	
  </dd>

  <dt>2. Website Construction</dt>
  <dd>
	<p>When constructing a website using a mobile-first design, the CSS located outside of the media queries is considered global in nature. Therefore, this CSS is applied to any type of device, as it is loaded <em>first</em>.</p>
	
	<p>Any media queries that are placed in the CSS stylesheet, are used to override these global settings. In a mobile-first design environment, typically these media queries are there to adjust layouts for larger dimensions &mdash; such as tablets, desktops, and larger screens. If the webpage is being viewed on a device that meets any of the media query parameters in the stylesheet, the CSS defined in that query will be applied. This mobile-first structure prevents elements from loading that might clog up a mobile display or load slowly due to a mobile device's limited processing power.</p>
	</dd>
</dl>

<li>Some techniques, like creating responsive tables with CSS, work better from a desktop-first perspective.</li>

    const regex = /(<link\b.+href=")(?!http)([/styles.css]*)(".*>)/gm;
    const cssContent = cssBlock.getContent();

    html= html.replace(regex, content);

function replaceJsScripts(html, jsBlock) {
    const regex = /(<script\b.+src=")(?!http)([/scripts.js]*)(".*>)/gm;
    const start ='<';
    const end=">";
