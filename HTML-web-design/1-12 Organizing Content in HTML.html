<p>This assignment discusses how to organize content in HTML using heading tags, paragraph tags, and both line and horizontal rule breaks.</p>

<h4>Headings</h4>

<p>Headings add order and structure to the content of a webpage. Headings can be used to highlight important topics within a webpage for users. Headers are also used by search engines to index the content of a webpage. Therefore, it is important to consider where and how headings are placed carefully.</p> 
  
<p>HTML offers six heading tag levels from &lt;h1&gt; to &lt;h6&gt;. These headings define a hierarchy, with the heading level number indicating the header's relationship to other headers in the hierarchy (think outline) of the page. The heading tag &lt;h1&gt; defines the primary heading in the document (usually the title), and there can be only one &lt;h1&gt; per page. An &lt;h2&gt; is the next level of heading in an HTML page. If your page has subheadings, the next level down <em>has</em> to be an &lt;h2&gt;. You can't skip the &lt;h2&gt; heading and put an &lt;h3&gt; in there instead (for example, just because you like the size of the &lt;h3&gt; heading better).</p>

<p>Always remember that HTML describes the structure of the page. So, if you have a &lt;h1&gt; for the page title followed by some &lt;h3&gt;s, there is something wrong with the page's structure.</p>

<p>Therefore, &lt;h3&gt;s are subheadings that belong under &lt;h2&gt;s, and &lt;h4&gt;s are subheadings that belong under &lt;h3&gt;s. Although HTML supports six levels of headings, you'll probably find in practice that you almost never use a level beyond &lt;h4&gt;.</p>

<p>By default, all browsers display heading in a different font style than the other text on the page. The default style is defined by the browser being used. In general, the heading font style is larger and bolder, with &lt;h1&gt; displayed in the largest size and &lt;h6&gt; headings displayed in the smallest size. It is worth mentioning that the  appearance of heading tags can be customized using the CSS font styles property &mdash; which is not covered in this course.</p>

<h4>Use of Headings</h4>

  <p>Do not use heading tags to style text &mdash; to make it larger or bolder, for example. Headings are only used to describe the structure of your HTML document.</p> 
  
  <p>The &lt;h1&gt;, indicating the primary level of heading, is placed at the top of an HTML document. HTML documents may have only one &lt;h1&gt; heading.</p>
  
  <p>Heading tags should appear in a descending order &mdash; i.e. &lt;h1&gt; followed by &lt;h2&gt; and so on.</p>
  
  <p>Search engines rely on the levels and order of headings to index the information on a page.</p>

<h4>Horizontal Lines and Line Breaks</h4>

<p>The &lt;hr&gt; (horizontal rule) tag visually breaks up content or sections of a webpage. This may seem as though it is presentational, however the correct use of the &lt;hr&gt; element is to signal a thematic break, or a change in topic, in an HTML page.</p>

<p>Similarly, the line break tag, or &lt;br&gt;, can be used to insert a single line break on a webpage. It is important to note that, similar to using empty &lt;p&gt; tags, it is bad form to use multiple &lt;br&gt; tags to insert vertical space in an HTML file. CSS must be used to create and manage the vertical spacing around HTML elements. The correct use of &lt;br&gt; is to create a line break among lines enclosed in a block-level element like &lt;address&gt;&lt;/address&gt;, for example.</p>
