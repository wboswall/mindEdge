<h4>Relative Links</h4>

<p><a role="button" tabindex="0" class="glossary-alert" id="glossTerm7" data-term="relative links" name="relative links" title="A link that points to a location that occurs in the same directory as the referring page">Relative links<span class="screenreader">*</span></a><span style="display:none;">&nbsp;A link that points to a location that occurs in the same directory as the referring page&nbsp;</span> (or <b>local links</b>) resolve to local resources (files): they are located somewhere within the same directory structure relative to the HTML file that has the link.  A relative link never includes the http:// or https:// prefix followed by a domain name. For example, a relative link looks like this:</p>
  
  <div class="medium yellow highlightbox; centerdiv">
	<p>&lt;a href="filename.html"&gt;</p>
</div>
	
<p>And assumes that "filename.html" is present in the same directory as the current file. This code syntax is illustrated in the example below.</p>

<img class="img-center" style="max-width:50em !important;" src="https://cdn-d.mindedgeonline.com/1675/relative_link_ex.png" alt="Screenshot of a relative link for a file located within the same folder"/>



<p>If you want the relative link to point to a file in a folder that is found one level above the current folder, you'll need to add two periods and a slash("../"). For example:</p>

<div class="medium yellow highlightbox; centerdiv">
  <p>&lt;a href="../filename.html"&gt;</p>
</div>
  
<p>The syntax above tells us that "filename.html" lives in a directory that is one level above the file that contains the link. This code syntax is illustrated in the example below.</p>

<img class="img-center" style="max-width:50em !important;" src="https://cdn-d.mindedgeonline.com/1675/relative_link_ex2.png" alt="Screenshot of a relative link for a file located one level above the current folder"/>



<p>If you want a link to point to a file that is in a different directory, but at the same level within the directory structure as the containing directory of your referring file, the directory name must be placed prior to the filename. For example, if the target file is in a directory named "pages," the href would be:</p>  
  
<div class="medium yellow highlightbox; centerdiv">
  <p>&lt;a href="../pages/filename.html"&gt;</p>
</div>
  
<p>This code syntax is illustrated in the example below.</p>

<img class="img-center" style="max-width:50em !important;" src="https://cdn-d.mindedgeonline.com/1675/relative_link_ex3a.png" alt="Screenshot of a relative link for a file located in a different folder within the same directory"/>



<p>Lastly, if you want to point to a file in the <a role="button" tabindex="0" class="glossary-alert" id="glossTerm8" data-term="root folder" name="root folder" title="The folder that is the highest directory in the file hierarchy. Also called the root directory">root folder<span class="screenreader">*</span></a><span style="display:none;">&nbsp;The folder that is the highest directory in the file hierarchy. Also called the root directory&nbsp;</span> of your website's files, the href attribute must begin with a "/." For example:</p>

<div class="medium yellow highlightbox; centerdiv">
  <p>&lt;a href="/filename.html"&gt;</p>
</div> 
 
<p>The syntax above is a link to a file named "filename.html" in the root folder. This example is illustrated below.</p>


<img class="img-center" style="max-width:50em !important;" src="https://cdn-d.mindedgeonline.com/1675/relative_link_ex4.png" alt="Screenshot of a relative link for a file located within the root folder"/>





<h4>Setting Targets for Links</h4>

<p>By default, all links will open in the current window or tab when it is clicked. By adding a target attribute, we can tell the browser where to open the linked document. If you want the link to open in a new browser window or tab, you can specify the link to open in a new browser tab as follows;</p> 
  

<div class="medium yellow highlightbox; centerdiv">
  <p>&lt;a href="http://skyelearning.com" target="_blank" &gt;</p>
</div>

<p>For example, &lt;a href="http://skyelearning.com" target="_blank"&gt; will open the Skye Learning homepage in a new window or tab. This allows the user to click on a link and reference it later without leaving the current page that they are viewing. This code syntax is illustrated in the example below.</p>

<img class="img-center" style="max-width:50em !important;" src="https://cdn-d.mindedgeonline.com/1675/setting_target_ex.png" alt="Screenshot of setting a target with a link"/>

<p>This is common practice when you provide a link that you believe is useful to your readers, but you don't want them to leave your website to open the link.</p> 

<p>The most common reason to use target="_blank" is that it keeps users on your site longer, improving site metrics such as <a role="button" tabindex="0" class="glossary-alert" id="glossTerm3" data-term="bounce rate" name="bounce rate" title="An internet marketing term that represents the percentage of website visitors who enter a website and then leave without continuing to view other pages within the same website">bounce rate<span class="screenreader">*</span></a><span style="display:none;">&nbsp;An internet marketing term that represents the percentage of website visitors who enter a website and then leave without continuing to view other pages within the same website&nbsp;</span>, <a role="button" tabindex="0" class="glossary-alert" id="glossTerm4" data-term="conversion" name="conversion" title="An internet marketing term that represents the percentage of users who take a desired action">conversion<span class="screenreader">*</span></a><span style="display:none;">&nbsp;An internet marketing term that represents the percentage of users who take a desired action&nbsp;</span>, and <a role="button" tabindex="0" class="glossary-alert" id="glossTerm6" data-term="pages visited" name="pages visited" title="An internet analytics term that represents the measure of how many pieces of content (pages) a specific user or group of users views on a single website">pages visited<span class="screenreader">*</span></a><span style="display:none;">&nbsp;An internet analytics term that represents the measure of how many pieces of content (pages) a specific user or group of users views on a single website&nbsp;</span>.</p>

<h3>Anchor Links</h3>

<p><a role="button" tabindex="0" class="glossary-alert" id="glossTerm1" data-term="anchor links" name="anchor links" title="A type of link that points to a location within the same page">Anchor links<span class="screenreader">*</span></a><span style="display:none;">&nbsp;A type of link that points to a location within the same page&nbsp;</span> go to another location within the same HTML document. Clicking on this type of link will make the browser scroll to a specific location on the same page. This is a useful navigation feature that allows users to skip large sections of a page. For example, anchors can be used to allow readers to conveniently navigate different locations on a page, such as back to the top or to an area further down the page.</p>

<p>Creating an anchor is a two-step process:</p>
<ul>
  <li>
    <p>First, you need to identify the targets (usually some headings) where you want the browser to go when the link is clicked. This is accomplished by adding an identifying id attribute value to each target element.</p>
  </li>
  <li>
	<p>Next, in the location where you want to place the link on the page, use the same identifying id attribute preceded by the hash sign (#) as the value of the href attribute of the opening &lt;a&gt; tag.</p>
  </li>
</ul>
	  



<p>The two parts work together with the anchor serving as a marker that identifies where the link associated with the anchor will take the user on the page when they click on the link.</p>

<p>The syntax for both the anchor and the link that points to the anchor is outlined in the table below.</p>

<table class="default_tbl4">
<tr>
    <th>&nbsp;</th>
  <th>Syntax</th>
</tr>
<tr>
    <th class="table_side">Anchor:</th>
    <td>&lt;a id="name of location where you want the link to point"&gt;&lt;/a&gt;</td>
</tr>
  <tr>
    <th class="table_side">Link to an anchor:</th>
    <td>&lt;a href="#name of location where you want link to point"&gt;content about the link that user will see&lt;/a&gt;</td>
</tr>
</table>

<h5>Example</h5>

<p>A common example of the use of an anchor is the "back to top" link that is often placed at the bottom of webpages. The syntax used to create this type of anchor link is as follows.</p> 

<p>First, we need to create an identifying name for the anchor. In this example, we will use the name top. So, we would place the word top inside the quotation marks as follows:</p>

<div class="medium yellow highlightbox centerdiv">
  <p>&lt;a id="top"&gt;&lt;/a&gt;</p>
</div>

<p>Next, we need to create the link that will point to the anchor on the page. To do this, we use a hashtag (#) followed by the id name we just created&mdash; in this example, the word top. Then, it is important to add language that directs the user where the link will take them when they click on it. For this, we will use the phrase "back to top." The syntax for this link is illustrated below:</p>

<div class="medium yellow highlightbox centerdiv">
  <p>&lt;a href="#top"&gt;Back to top&lt;/a&gt;</p>
</div>

<p>The graphic below illustrates how the code looks and renders on a page. Notice the "back to top" link that appears at the bottom of the webpage. </p>

<img class="img-center" style="max-width:50em !important;" src="https://cdn-d.mindedgeonline.com/1675/anchor_element_code_ex.png" alt="Screenshot of examples of the basic syntax of an anchor and anchor link. The Anchor: a id=top is placed at the top of the html document. The anchor link stating 'back to top' is placed at the bottom of the content within the body tags">

    <p>To create an ID link that points to a specific id on a different page, add the hash symbol (#) and name of the ID attribute to the end of the URL of the target location (assuming there is an id attribute with the same value on the other page).</p>

class CodeBlockClass {

    constructor(blockIndex, type, debug) {
        this.container = '';
        this.index = '';
        this.options = {
            mode: "htmlmixed",
            styleActiveLine: false,
            lineNumbers: true,
            lineWrapping: true,
            theme: "rubyblue",
            autoRefresh: true,
            matchBrackets: true,
            showHint:false,
            indentWithTabs: true,
            autoCloseTags: {
                whenOpening: false,
                whenClosing: true
            },
            value: ''
        };
        let sectionTag = 'codeSection';
        this.index = blockIndex;
        switch (type) {
            case 'html':
                this.options.mode = 'htmlmixed';
                this.options.theme = 'rubyblue';
                sectionTag = 'htmlSection';
                break;
            case 'css':
                this.options.mode = type;
                this.options.theme = 'rubyblue';
                sectionTag = 'cssSection';
                break;
            case 'js':
                this.options.theme = 'rubyblue';
                this.options.mode = 'javascript';
                sectionTag = 'codeSection';
                break;
        }
        if(debug) {
            this.options.mode= 'html';
        }
        this.container = document.getElementById(`${sectionTag}-${blockIndex}`);
    }


    init() {
        this.editor = CodeMirror.fromTextArea(this.container, this.options);
        this.editor.setSize(null, 400);
    }

    getContent() {
        return this.editor.getValue();
    }

    setContent($content) {
        this.editor.setValue($content);
    }

    updateTheme(theme) {
        this.editor.setOption("theme", theme);
    }

   refresh() {
       this.editor.refresh();
    }

}

function resetCodeBlock(codeBlock) {
    if(codeBlock){
        Object.keys(codeBlock.blocks).forEach(type => {
            let content = $(`.${type}-reset-container-${codeBlock.section}`).val();
            if(type==='html'){
                const regex = /(<link\b.+href=")(.*[/[styles].css]*)(".*>)/gm;
               content = content.replace(regex, '<link rel="stylesheet" href="./styles.css"/>');
            }
            codeBlock.blocks[type].setContent(content);
        })
        codePreview(codeBlock)
    }

function toggleType(section, type){
    const sectionContainer =$(`#codingSection-${section}`);
    const htmlSection = sectionContainer.find(`#html`);
    const cssSection = sectionContainer.find(`#css`);
    const jsSection = sectionContainer.find(`#js`);
    if(lastType[section] === type){
        return;
    }
    lastType[section] = type;
    switch (type) {
        case 'html':
            htmlSection.toggleClass('active');
            cssSection.removeClass('active');
            jsSection.removeClass('active');
            break;
        case 'css':
            cssSection.toggleClass('active');
            htmlSection.removeClass('active');
            jsSection.removeClass('active');
            break;
        case 'js':
            jsSection.toggleClass('active');
            cssSection.removeClass('active');
            htmlSection.removeClass('active');
            break;
    }
    codeBlocks[section].blocks[type].refresh();
}


function updateFrame(section, html) {
    const previewFrame = document.getElementById(`preview-content-${section.section}`);
    const preview =  previewFrame.contentDocument ||  previewFrame.contentWindow.document;
    preview.open();
    preview.write(html);
    preview.close();
}

function replaceStyles(html, cssBlock){
    const regex = /(<link\b.+href=")(?!http)([/styles.css]*)(".*>)/gm;
    const cssContent = cssBlock.getContent();
    const content = `<style> ${cssContent} </style>`;
    html= html.replace(regex, content);
    return html;
}

function replaceJsScripts(html, jsBlock) {
    const regex = /(<script\b.+src=")(?!http)([/scripts.js]*)(".*>)/gm;
    const start ='<';
    const end=">";
    const jsContent = jsBlock.getContent();
    const content = `${start}script${end} ${jsContent} ${start}/script${end}`;
    html= html.replace(regex, content);
    return html;
}

function codePreview(section){
    let html = section.blocks.html.getContent();
    if(section.blocks.css){
        html = replaceStyles(html, section.blocks.css);
    }
    if(section.blocks.js){
        html = replaceJsScripts(html, section.blocks.js);
    }
    updateFrame(section, html);
}

        contentId: contentId,

          console.log(response);
          button.removeAttr("disabled");
        }
    });
}
