Further, negative space not only exists around an element, but also inside an element. </p>

<p>Employing negative space in a web design can have a great impact on the following factors of user experience: </p>

<ul>
  <li><b>Content readability:</b> Using adequate space between lines, letters, words, and elements increases content readability and comprehension. For example, negative space can transform a long paragraph of text into short paragraphs or lists that are more inviting for the reader.</li>

  </ul>

  <p>A common misconception about negative space is that it requires less content to work. However, a text-heavy website can still leverage white space to create focus, drive the focus to important elements, and promote a feeling of sophistication and style.</p>

<p>When it comes to color and accessibility, the leading concern is contrast. Below are several key takeaways:</p>

<ul>
  <li>Provide sufficient contrast between the background and foreground</li>
  <li>Use color and something else to draw attention to a certain element; for example, you could increase the font weight of the text inside a call-to-action button</li>

<dt>Principle #4: Alignment</dt>

  <dd>
<img src="https://cdn-d.mindedgeonline.com/1671/cluttered_vs_aligned.png" alt="Example of a webpage that has misaligned elements and aligned elements." class="img-center" style="max-width:40em !important;">
 
<p><a role="button" tabindex="0" class="glossary-alert" id="glossTerm0" data-term="alignment" name="alignment" title="The positioning of elements on the same vertical, horizontal, or diagonal line.">Alignment<span class="screenreader">*</span></a><span style="display:none;">&nbsp;The positioning of elements on the same vertical, horizontal, or diagonal line.&nbsp;</span> is an integral part of an effective website design that most people forget about &mdash; until it's not there. Alignment acts as the backbone for design and allows the web designer to make conscious decisions about how and where to place elements on the webpage. </p>

<p>Alignment covers everything from larger elements, including the header, the content, and the navigation, to smaller elements, including paragraphs, sub-headings, and images. This assignment will cover how to align three fundamental page elements: text, images, and element groups.</p>

<h4>Text Alignment</h4>

<p>Text appears in multiple areas on a webpage, from the navigation menu to the header and body text. The key to aligning these different elements is to develop a consistent style within each text element type. For example, header text should always align to the left, or the navigation text should always align to the right.</p> 

<p>Consider how suitable each alignment is for different types of text. For instance, left aligned text is the default setting in most computer software. This is because those who read left-to-right often have difficulty reading long paragraphs of text that are aligned to the right. The same is true for long paragraphs of text that are aligned in the center. This is why left-aligned text is the standard for lengthy blocks of text.</p>

<img src="https://cdn-d.mindedgeonline.com/1671/misaligned_vs_aligned_to_margin.png" alt="Example of a webpage that has misaligned elements and aligned elements." class="img-center" style="max-width:40em !important;">

<p>Smaller amounts of text, on the other hand, can work well with a center alignment. Further, most website headers are aligned either to the left or center, and occasionally to the right. This is a great technique to create emphasis and draw the eye to a particular line of text. </p>


<h4>Image Alignment</h4>

<p>Consistency is an important factor in image alignment. For example, the space between an image and a block of text should remain consistent around the image. Further, there should be equal space between one image to the next. </p>

<img src="https://cdn-d.mindedgeonline.com/1671/aligned_image_vs_misaligned_image.png" alt="Example of a webpage that has misaligned elements and aligned elements." class="img-center" style="max-width:40em !important;">

<p>Like text elements, images can be aligned to the left, right or center. Typically, a display image in the header is centered for maximum impact. Or, it may align to the left or right depending on how it interacts with the text or other elements in the header area. </p>

<h4>Element Group Alignment</h4>

<p>Most websites are divided into sections that display groups of elements. For example, the elements in a header section may include a navigation menu, a logo, and a display image. The footer section may include contact information or additional links. These are element groups.</p>

<img src="https://cdn-d.mindedgeonline.com/1671/aligned_elements_vs_misaligned_elements.png" alt="Example of a webpage that has misaligned elements and aligned elements." class="img-center" style="max-width:40em !important;">

<p>Aligning the elements in each group adds another level of organization and overall cohesion to a website. For this reason, it is a good idea to establish margins and spacing with the help of gridlines. Gridlines ensure the elements inside each section do not bleed outside the intended margin&mdash;unless it's an intentional part of the design. </p>

<p>It is important to note that alignment is not organizing elements into perfect squares. Elements of different shapes and sizes can be aligned within its respected group. For example, smaller images can alternate between the left-side and right-side of a particular section. This is a great way to add emphasis to a particular element. </p>

    // auto save student response after 2 seconds of no changes
    function autoSave(noDelay) {
        var self = this;
        self.editor.save();

        //let's get this thing to adhere to the 6-second rule
        var d = new Date();
        var n = d.getTime();
        var diff = n - self.lastAutoSaveTime;

        if (self.autoSaveTimer !== null) {
            // reset the timer so it runs AUTO_SAVE_DELAY seconds after stopping typing
            clearTimeout(self.autoSaveTimer);
            self.autoSaveTimer = setTimeout($.proxy(autoSave, self), AUTO_SAVE_DELAY);
        }

        if (!self.hasNewContent()) {
            //so the very next keystroke doesn't trigger a save if it's been more than AUTO_SAVE_DELAY seconds
            self.lastAutoSaveTime = n;
            return;
        }

        if (diff < AUTO_SAVE_DELAY) {
            //text change not necessary here
            //self.$autosave.html(UP_TO_DATE_MSG);
            setWarning();
            return;
        }

        unsetWarning();


        self.lastAutoSaveTime = n;
        //end of 6-second enforcement

        if (noDelay === true) {
            // save automatically with no delay
            self.isAutosaving = true;
            self.$autosave.html(SAVING_MSG);
            self.autoSaveRequest();
        } else if (self.hasNewContent()) {
            self.isAutosaving = true;
            self.$autosave.html(SAVING_MSG);
            self.autoSaveRequest();
        } else {
            self.$autosave.html(UP_TO_DATE_MSG);
        }
        //self.autoSaveTimer = setTimeout($.proxy(autoSave, self), AUTO_SAVE_DELAY);
    }

    // just the request and handlers
    function autoSaveRequest() {

    }
