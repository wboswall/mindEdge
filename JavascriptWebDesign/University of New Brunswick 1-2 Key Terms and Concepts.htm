Key Terms and Concepts

This assignment covers a number of key terms and concepts followed by a set of flashcards that you can use to test yourself on your knowledge of some of the vocabulary related to JavaScript and web design.

Ecma (formerly the European Computer Manufacturers Association)

In 1997, Ecma* Ecma is a standards organization that was formerly known as the European Computer Manufacturers Association. In 1994, it changed its name to its acronym, and thus today, its preferred spelling is Ecma. To further complicate matters, its standards (e.g. ECMAScript) tend to capitalize ECMA. In addition to JavaScript (technically ECMAScript), Ecma also has standards related to the C++ programming languages and XML.  adopted the first ECMAScript standard. While it has gone through several iterations since it remains the standard upon which JavasScript is based. For those unfamiliar with such standards, think of ECMAScript as a combination of Warriner's English Grammar and the Chicago Manual of Style; it dictates the conventions of how a script interpreter should expect and respond to the instructions sent it. In that regard, it also dictates how a script should be written.

Index

In the world of most web servers, the word "index" carries special meaning. In most cases, a file named index becomes the default file served if none is specified. For example, www.example.com/folder/index.html results in the same content as www.example.com/folder. A few things to watch out for, however:

  • Not all web servers follow this convention. For example, for years, Microsoft servers preferred "default" over "index" as the naming convention. This can be especially problematic if your site moves from one platform to another, or there is a difference between development and production environments.

  • There is an order of preference. If you have four files (e.g., index.html, index.htm, index.php, index.asp) all in a directory, in the server's configuration, there is an order of selection.

  • Not all servers or directories on servers are configured like this. In some cases, the default behavior for a URL that does not end in a file name is to simply display a listing of all the files in the directory (i.e., an index of the folder's contents), and this can pose a certain security hazard. That said, you should never upload something to the Internet-facing side of a web server that you intend to keep from prying eyes. While there are certain things you can do (precede the file name with a "." or ensure nothing links to it), these are not guarantees.

Libraries and Frameworks

In JavaScripting, we hear the terms "framework" and "library" a lot. In many ways you can equate the terms even though there is a distinction. A framework provides a structure or environment for creating an application. A library, on the other hand, is a set of routines that can be used anywhere throughout a site. A non-technical analogy is that a library is like using an active voice in writing, and a framework is like using the passive one.

In the case of the library, a developer writes code occasionally calling routines from the library; the library fills the gaps for the developer. However, in a framework, the framework is at the center, and at certain points, the developer supplies the extra code necessary. With a framework, the developer fills the gaps.

In technical terms, a framework represents Inversion of Control* Inversion of Control is a programming principle that describes a main program handing overall control to a sub-routine or program.  (IoC), a software engineering principle where the control of parts of a program is transferred out of the main program.

In the overall scheme, a library might require more work but provide more flexibility, but a framework may be more ordered.
